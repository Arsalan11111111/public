Lims Sale

New models:

  1. Analysis Costing (appears in Analysis --> notebook "Costing")
    • Product: Product used in the analysis
    • Qty: The quantity of product
    • Cost: Cost of the product
    • Revenue: Revenue of the product (Automatically computed when you select a product)

New fields:

  1. Analysis
    • Ref PO: Reference of a Purchase Order
    • Order: Sale Order
    • Costing: (See section Analysis Costing)
    • Total Cost: Total cost of all "Analysis Costing" of the Analysis
    • Total Revenue: Total revenue of "Analysis Costing" of the Analysis
  2. Analysis Request
    • Pricelist: It's the pricelist
    • Sale Order: Sale Order
    • Amount Untaxed: it's the amount untaxed of the sale order
  3. Analysis Request Sample
    • Sale orders: Sale Order
  4. Parameter Characteristic
    • Billable: Box checked if the product is billable
    • Product: (Appears if billable is checked) Product used in the Parameter
    • Sales Price: (Appears if billable is checked) Price of the product (related to list price of the product)
  5. Product Template
    • Lims For Sales: Box checked if this product can be select
  6. Partner
    • Type: add 1 selection (Labo) in type
  7. Sale Order
    • Request: (See lims base --> section Analysis Request)
    • Customer contacts:
    • Signatory: Person who signed the sale order
    • Analysis: (See lims base --> section Analysis)
    • Customer Ref: Reference of the customer
  8. Sale Order Line
    • Sample: (See lims base --> section Analysis request sample)
    • Parameter: (See lims base --> Parameter Characteristic)
    • Matrix: (See lims base --> section Matrix) (field Automatically fill with the matrix of the parameter)
