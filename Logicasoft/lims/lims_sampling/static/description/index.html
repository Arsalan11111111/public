Lims sampling point

New models:

  1. Quality zone
    • Code: free text, short identifier
    • Entity: Create a link to a contact
    • Sampling Point: list of sampling points depending on the quality zone; used for the analysis buttons (smart buttons) on the quality zone
    • Parent: A quality zone; organizes the quality points in a tree structure
  2. Sampling Point
    • Code: free text, short identifier
    • Matrix: The sampling point and the analysis must have the same matrix for the link to be effective.
    • Customer: Link to a contact
    • Quality Zone: link to a default quality zone; adds the quality zone defined by default when adding the sampling point on an analysis
    • Location: link to a default sampling point location; adds the quality zone defined by default when adding the sampling point on an analysis
    • Sampling Category (O2M)
    • Sampling Type (O2M)
    • Owner: Link to a contact
    • Tags (M2M)

New fields:

  1. Analysis Request (only on samples)
    • Sampling Point (M2O): defines the sampling point when creating the analysis
  2. Analysis
    • Sampling Point (M2O)
    • Location (M2O)
    • Sampling type (M2O)
    • Quality zone (M2O)
  3. Sop (tests)
    • Sampling Point (M2O): related to Sampling Point define in analysis
  4. Result(s) (all types)
    • Sampling Point (M2O): related to Sampling Point define in analysis

New features:

  • Allows to manage the sampling points in the Lims.
  • It is possible to group the analyses on sampling points.
  • It is possible to manage quality zones in the Lims.
  • See elements in menu(s) :
    • Lims > Master data > Sampling point
    • Lims > configurations > Sampling point
